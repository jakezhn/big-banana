# Web Docs

`apps/web/docs/` 专门服务于网站内容本身，而不是后端工程设计。

## 这里应该放什么

- 页面信息架构
- 页面模块草图
- 页面文案与语气规范
- 用户引导内容
- FAQ / 帮助中心草稿
- legal / disclosure / risk notice 页面规划
- dashboard 页面内容约束

## 这里不应该放什么

- 数据库设计
- agent / risk / execution 架构
- Supabase / Vercel / Inngest 部署方案
- schema 冻结规格
- 后端联调 runbook

这些内容应继续留在根目录的 `docs/development/`。

## 建议的后续结构

- `information-architecture.md`
- `dashboard-content-model.md`
- `marketing-pages.md`
- `help-center.md`
- `legal-pages.md`

当前阶段还没有正式的网站内容体系，所以这里只先建立边界，不提前写空文档。
